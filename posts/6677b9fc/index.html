Github pages搭建博客

前排提示：不少内容直接搬运自瑾知的博客

Git安装

前往Git官网，点击Downloads下载。

选择对应的操作系统。我用的是Windows。

全部默认选择，Next即可。

摆烂了，直接用瑾知的图了，git最新版是2.36，其他没区别

点击Git Bash Here进入Bash命令行。在桌面右键就是进入桌面所在的目录。输入git回车，得到第二张图的提示则代表Git安装成功。你也可以输入git --version查看版本号检查是否安装成功。


GitHub绑定

创建GitHub仓库

首先，你需要一个GitHub账号，如果没有需要先注册一个。如果GitHub无法访问或速度过慢，可以尝试科学上网。

登录GitHub后，创建一个新的仓库。

也用瑾知的了，过程一样的

Repository name可以设置为和用户名同名，或者设置为*你的用户名*.github.io，这样设置的话你的博客默认地址不带二级目录，可以直接通过*你的用户名*.github.io访问。能这样做的原因是，github里面与自己用户名重名的仓库比较特殊，由此生成的github pages会成为你的对应个人主页；而命名为*你的用户名*.github.io则是直接创建了你的个人主页的仓库。（与自己用户名重名的仓库里的readme.md文件也比较特殊，可以自己改改试试，看看github个人主页有啥变化。）

注意两个方法只能选一个，比如对于我，如果我先创建一个仓库名字叫chaoswinder.github.io，再创建一个仓库名字叫chaoswinder，那么我访问前者的pages，默认地址是chaoswinder.github.io，而后者是chaoswinder.github.io/chaoswinder 。（如果顺序反过来，前者的默认地址会更加离谱：chaoswinder.github.io/chaoswinder.github.io ，后者则是变回了chaoswinder.github.io。应该是为了避免重复吧。）另外github仓库名对大小写不敏感（就是大小写都当一个字母看），所以你可以写小写的用户名。

你也可以写别的名字，比如blog，那么访问地址就会是*你的用户名*.github.io/blog。

设置用户名和邮箱，–global 为全局参数，表明本地所有Git仓库都会使用这个配置。注意代码里面的“yourname”“your_email”之类的东西要换成你自己的相关信息，很多教程都会这样写，之后看到记得替换。

创建一对SSH密钥，其实-C后面的东西可以随便输（代表comment），一般习惯是输入邮箱。

之后回车默认即可，会在User文件夹对应的用户下创建.ssh文件夹，去你自己用户名的文件夹下面寻找。

你会找到上图的id_rsa和id_rsa.pub两个文件，前者是私钥，后者是公钥。如果你只看到两个id_rsa文件，其中一个还是publisher文件，应该是因为你的电脑预装了Microsoft Publisher，同时设置了隐藏文件后缀（扩展名）。点击上方的查看，勾选“文件扩展名”，你就能看到哪个是.pub文件了。

vscode（没有就用记事本）打开id_rsa.pub文件，复制生成的公钥，找到github设置里面的settings；

然后点击SSH and GPG keys -> New SSH key，将id_rsa.pub的内容粘贴到Key下面的框中，Title可以随便填，你可以随便写个名字以便记住它的用途。


最后点击Add SSH key。之后可以在git Bash中输入ssh -T git@github.com进行检验，出现下图信息则表示绑定成功。
