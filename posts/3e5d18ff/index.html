220319 D - Fault-tolerant Network

发表于2022-03-21|更新于2022-03-28|算法竞赛
|字数总计:1k|阅读时长:5分钟|阅读量:

题目

There is a classroom with two rows of computers. There are nn computers in each row and each computer has its own grade. Computers in the first row has grades a1,a2,…,an and in the second row — b1,b2,…,bn.
Initially, all pairs of neighboring computers in each row are connected by wire (pairs (i, i+1) for all 1≤i<n), so two rows form two independent computer networks.
Your task is to combine them in one common network by connecting one or more pairs of computers from different rows. Connecting the i-th computer from the first row and the j-th computer from the second row costs |ai - bj|.
You can connect one computer to several other computers, but you need to provide at least a basic fault tolerance: you need to connect computers in such a way that the network stays connected, despite one of its computer failing. In other words, if one computer is broken (no matter which one), the network won’t split in two or more parts.
That is the minimum total cost to make a fault-tolerant network?

Input
The first line contains a single integer tt (1≤t≤10^4 ) — the number of test cases. Next tt cases follow.
The first line of each test case contains the single integer nn (3≤n≤2⋅10^5 ) — the number of computers in each row.
The second line contains nn integers a1,a 2,…,a n(1≤ai≤10^9 ) — the grades of computers in the first row.
The third line contains nn integers b1,b2,…,bn(1≤bi≤10^9 ) — the grades of computers in the second row.
It’s guaranteed that the total sum of nn doesn’t exceed 2⋅10^5.
Output
For each test case, print a single integer — the minimum total cost to make a fault-tolerant network.

Sample 1
Input
2
3
1 10 1
20 4 25
4
1 1 1 1
1000000000 1000000000 1000000000 1000000000
Output
31
1999999998

Note
In the first test case, it’s optimal to connect four pairs of computers:
computer 1 from the first row with computer 2 from the second row: cost∣1−4∣=3;
computer 3 from the first row with computer 2 from the second row: cost∣1−4∣=3;
computer 2 from the first row with computer 1 from the second row: cost ∣10−20∣=10;
computer 2 from the first row with computer 3 from the second row: cost ∣10−25∣=15;
In total, 3+3+10+15=31.
In the second test case, it’s optimal to connect 1 from the first row with 1 from the second row, and 4 from the first row with 4 from the second row.

思路

算是思维题，一开始没明白这个容错网络是个啥意思，后来懂了之后发现这题居然可以用枚举做
两行计算机如果要能保证任意一台计算机出故障之后仍然能连接成一个网络，那么每一行首尾两端的计算机一定要连上另外一行（无论哪个节点）；因为如果不连接它们的话，一旦该行最左边/最右边连接到另一行节点出故障，那两侧的计算机链就无法连接上整个网络了；而连接上它们，即使首尾中任何一台计算机出故障，剩余的计算机也可以走另外一边连上网。而保证了首尾连接之后，中间的节点就可以不用连接了。因此可以枚举首尾节点到另外一行计算机的所有距离，寻找其中最短的，然后把各种可能的组合全部尝试一遍即可。注意枚举的时候仍然要带上另一行的首尾（我最开始认为首尾连接单独列出了，没有枚举首尾，然后WA），比如a行3 10000 5，b行1 10000 3，连线会变成N字型，不扫首尾就会错。
